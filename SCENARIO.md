# Сценарии бота Zalivai

> **Правило:** КАЖДЫЙ экран содержит кнопки навигации. Ни одного экрана без кнопок.

---

## Универсальный сценарий

### Шаг 1. /start — Приветствие

**Триггер:** пользователь нажал `/start`

**Проверка:** `is_registered = TRUE/FALSE`

```
[Баннер + caption]
"Система массовой автоматизированной генерации профессионального
фото и видео контента для товаров с последующим размещением в социальных сетях.

Возможно создавать фото и видео в различных форматах по заранее спроектированным промптам для ваших товаров."

[Inline: Начать →]
```

**Переход:** Нажал «Начать →» → Шаг 2

  📂 Мои эталоны
  💰 Пополнить баланс

### Шаг 4. Ввод артикула

```
[Баннер + caption]
"В строку сообщений введите артикул.

Мы загрузим фото из карточки. Выберите
3 лучших — где ваш товар виден наиболее
чётко и детально. Это станет основой
для генерации фото и видео контента."

[Inline: ← Назад (к маркетплейсу) | 🏠 Меню]
```

### Шаг 6. Показ карточки товара

```
[Баннер + ПЕРВОЕ фото товара из WB + caption]
"Найдено:

📦 {name}
🏷 Бренд: {brand}
🎨 Цвет: {color}
🧵 Материал: {material}

То что мы хотели?"

[Inline: ✅ Да | ❌ Нет]
[Inline: ← Назад (к маркетплейсу) | 🏠 Меню]

**Данные из wb_parser:**
- `name` — название товара
- `brand` — бренд

- `images[0]` — первое фото товара

**Переходы:** (4 кнопки по 2 в стороке, последовательность приоритетов сверху вниз, слева направо)
- «❌ Нет» → Шаг 4 (повторный ввод артикула)
- «✅ Да» → Шаг 7
- «← Назад» → Шаг 3
- «🏠 Меню» → Шаг 2

### Шаг 8. Предложение создать эталон

```
[Баннер + caption]
"Создаю эталон — чистое фото товара на прозрачном
фоне без лишних предметов.

Этот эталон будет использоваться для генерации
фото и видео контента.

💰 Стоимость: {price}₽
💳 Ваш баланс: {balance}₽

Создать эталон?"

[Inline: ✅ Создать эталон]
[Inline: ❌ Отмена | ← Назад (к карточке) | 🏠 Меню]
```

**Действия:**
- Читаем прайслист: `price = get_price('create_reference')`
- Читаем баланс пользователя: `balance = get_user_balance(user_id)`
- Если `balance >= price` → ждём подтверждения
- Если `balance < price` → Шаг 8а (недостаточно средств)

**Переходы:**
- «❌ Отмена» → Шаг 4
- «← Назад» → Шаг 6 (или Шаг 7б-1 если артикул существующий)
- «🏠 Меню» → Шаг 2
- «✅ Создать» → Шаг 9

---

→ Удаляется через 5 секунд
```

**Что видит пользователь после удаления алерта:**
```

[Inline: ← Назад | 🏠 Меню]
```

**Переходы:**
- «💰 Пополнить» → Flow оплаты
- «← Назад» → Шаг 8
- «🏠 Меню» → Шаг 2

---

### Шаг 9. Генерация эталона

```
[Баннер + caption]
"⏳ Генерирую эталон..."

→ Удаляется сразу после завершения генерации
```

**Действия (в фоне):**
1. `save_article(...)` — сохраняем товар в БД
2. `deduct_balance(user_id, price)` — списываем деньги
3. T2T API → генерируем промпт + категорию товара
4. I2I API → генерируем эталонное фото (фото товара + промпт → чистый фон)
5. Скачиваем результат, загружаем в Telegram, получаем `file_id`

**Переход:** Эталон готов → Шаг 10

---

### Шаг 10. Утверждение эталона

```
[Баннер + фото эталона + caption]
"🎉 Эталон готов!

📦 Артикул: {articul}
📸 Это ваш {ref_number}-й эталон для этого товара

Проверьте качество:"

[Inline: ✅ Подтвердить | 🔄 Перегенерировать (-{price}₽)]
[Inline: ← Назад (к выбору) | 🏠 Меню]
```

**Переходы:**

#### 10а. 🔄 Перегенерировать
- Проверяем баланс
- Если `balance >= price` → списываем → Шаг 9 (новая генерация)
- Если `balance < price` → Шаг 8а (недостаточно средств)
- Новый эталон получает `ref_number = N + 1`

#### 10б. ✅ Подтвердить → Шаг 11 (сохранение)

- «← Назад» → возврат к Шагу 8
- «🏠 Меню» → Шаг 2

---

### Шаг 11. Сохранение эталона

**Действия:**
```
article_references:
  - user_id
  - articul
  - reference_number = COUNT(active refs for this articul) + 1
  - file_id (telegram file id)
  - file_path (локальный путь)
  - reference_image_url (публичный URL для I2I API)
  - category (из T2T: верх/низ/обувь/головной убор)
  - reference_prompt (английский промпт из T2T)
  - is_active = TRUE
```

## Справочник

### Типы сообщений

| Тип | Баннер | Авто-удаление | Описание |
|-----|--------|---------------|----------|
| **Экран** | ✅ Да | ❌ Нет | Основной экран с информацией |
| **Промежуточный** | ✅ Да | ✅ Да (сразу) | «⏳ Загружаю...», «⏳ Генерирую...» |
| **Алерт** | ❌ Нет | ✅ Да (5 сек) | Ошибки, предупреждения |

### Сущности БД

| Таблица | Описание |
|---------|----------|
| `users` | Пользователь (balance, is_registered) |
| `articles` | Распарсенные товары (name, color, material, wb_images) |
| `article_references` | Эталоны (N на артикул, ref_number, is_active) |
| `generation_tasks` | Очередь задач (photo/video, status, result_url) |
| `marketplace_cache` | Кэш маркетплейса по артикулу |
| `user_actions` | Лог действий пользователя |

### Прайслист (пример)

| Действие | Цена |
|----------|------|
| Создание эталона | 50₽ |
| Генерация фото | 30₽ |
| Генерация видео | 80₽ |
| Перегенерация эталона | 50₽ |

### Категории товаров

- `верх` — рубашка, футболка, блузка, свитер, куртка
- `низ` — юбка, брюки, шорты, джинсы
- `обувь` — кроссовки, туфли, сапоги
- `головной убор` — шапка, кепка, шляпа
