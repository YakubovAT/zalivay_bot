# Сценарии бота Zalivai

> **Правило:** КАЖДЫЙ экран содержит кнопки навигации. Ни одного экрана без кнопок.

---

## Универсальный сценарий

### Шаг 1. /start — Приветствие

**Триггер:** пользователь нажал `/start`

**Проверка:** `is_registered = TRUE/FALSE`

```
[Баннер + caption]
"Система массовой автоматизированной генерации профессионального
фото и видео контента для товаров с последующим размещением в социальных сетях.

Возможно создавать фото и видео в различных форматах по заранее спроектированным промптам для ваших товаров."

[Inline: Начать →]
```

**Переход:** Нажал «Начать →» → Шаг 2

### Шаг 4. Ввод артикула

```
[Баннер + caption]
"Шаг 4: Ввод артикула

В строку сообщений введите артикул.

Мы загрузим фото из карточки. Выберите
3 лучших — где ваш товар виден наиболее

для генерации фото и видео контента."

[Inline: ← Назад | 🏠 Меню]
```

**Переход:** Пользователь ввёл текст (цифры или ссылку, нужно вычленить из ссылки артикул, он как правило 6-9 цифр подряд).

### Окно скачивания (временное)

```
[Баннер + caption]
"⏳ Ищу товар...1"
(через 1 сек)
"⏳ Ищу товар...2"
(через 1 сек)
"⏳ Ищу товар...3"
...
(максимум до 15)
→ Это ВРЕМЕННОЕ сообщение удаляется сразу после ответа от wb_parser
```

**Особенность:**
- Это **отдельное сообщение**, появляется поверх экрана Шага 4
- Экран Шага 4 **остаётся видимым** под ним
- Когда парсинг завершён → Окно скачивания **удаляется** (как будто его не было)
- Экран Шага 4 **перерисовывается** в экран Шага 5 (редактируется `edit_message_media`)

**Действия:**
- Вызов `wb_parser.get_product_info(articul)`
- Состав очищается от цифр и процентов: `"полиэстер 90%; вискоза 10%"` → `"полиэстер; вискоза"`
- Если товар найден → Шаг 5 (перерисовка Шага 4)
- Если товар НЕ найден → Шаг 4а (АЛЕРТ)

---

### Шаг 4а. Товар НЕ найден — АЛЕРТ

```
🔴 [БЕЗ баннера, обычный текст]
"❌ Артикул {articul} не найден. Проверьте и попробуйте снова."

→ Удаляется автоматически через 5 секунд
```

**Что видит пользователь после удаления алерта:**
```
[Баннер — тот же что и в Шаге 4]
"Шаг 4: Ввод артикула..."

[Inline: ← Назад | 🏠 Меню]
```

**Переход:** Пользователь вводит новый артикул → Окно скачивания

### Шаг 5. Показ карточки товара

> **Важно:** Этот экран получается **редактированием** экрана Шага 4, а не созданием нового. Окно скачивания к этому моменту уже удалено.

```
[Баннер + ПЕРВОЕ фото товара из WB + caption]
"Шаг 5: Найден товар

📦 {name}
🏷 Бренд: {brand}
🎨 Цвет: {color}
🧵 Состав: {material}

Это тот товар?"

[Inline: ✅ Да, это он | ❌ Нет, другой]
[Inline: ← Назад | 🏠 Меню]
```

**Данные из wb_parser:**
- `name` — название товара
- `brand` — бренд
- `color` — первый цвет из списка
- `material` — состав (без цифр и процентов)
- `images` — список URL всех фото товара (до 30 штук)

**Переходы:**
- «❌ Нет, другой» → Шаг 4 (повторный ввод артикула)
- «✅ Да, это он» → Шаг 6
- «← Назад» → Шаг 3
- «🏠 Меню» → Шаг 2

### Шаг 6. Выбор фото из карточки

> **Папка:** `media/{user_id}/WB/{articul}/` — создаётся при подтверждении товара. Скачиваются все фото (до 30 штук).
> **Важно:** Это **один экран**, который плавно меняется. Сначала появляется "Загрузка фото..." (5 сек с анимацией), затем он же превращается в экран выбора.

```
[Баннер + фото товара + caption]
"Шаг 6 из N: Выбор фото — 1 из 12

Выберите фото №1 — где товар виден лучше всего."

[Inline: ⚪ 1  ⚪ 2  ⚪ 3]
[Inline: ← Пред.  1/12  След. →]
[Inline: ← Назад | 🏠 Меню]
```

**Логика выбора:**

Нажал ⚪ 1 -> Выбралось, фото скрылось -> Переход к следующему свободному.
Нажал ⚪ 2 -> Выбралось, фото скрылось -> Переход к следующему свободному.
Нажал ⚪ 3 -> Выбралось, фото скрылось -> Переход к следующему свободному (или появление кнопки "Утвердить").
Нажал на уже выбранное (🔘 1) -> Выбор отменился, фото появилось в списке.
**Переходы:**
- «← Пред.» / «След. →» — листают фото (редактирование текущего экрана)
- `⚪ 1/2/3` — выбор фото в слот (кнопка становится `🔘 1/2/3`)
- «✅ Утвердить выбор» → Шаг 7 (показ коллажа)
- «← Назад» → Шаг 3
- «🏠 Меню» → Шаг 2

### Шаг 7. Создание эталона (Коллаж)

> На этом шаге бот **склеивает** 3 выбранных фото в одно изображение (горизонтальный коллаж).

```
[Баннер + КОЛЛАЖ из 3-х фото + caption]
"Шаг 7 из N: Создание эталона

Вы выбрали 3 фото для артикула 12345678.

Убедитесь, что на этих фото товар виден лучше всего —
по ним будет создан эталон для генерации контента."

[Inline: ✅ Создать эталон]
[Inline: ← Назад | 🏠 Меню]
```

**Действия:**
- 3 фото склеиваются в один файл `media/{user_id}/temp/{articul}_collage.png`.
- Выбранные фото и артикул сохранены в контексте для следующего шага.

**Переходы:**
- «✅ Создать эталон» → Шаг 8 (проверка баланса → генерация T2I)
- «← Назад» → Шаг 6 (к выбору фото)
- «🏠 Меню» → Шаг 2

### Шаг 9. Предложение создать эталон

```
[Баннер + caption]
"Шаг 9: Создание эталона

Создаю эталон — чистое фото товара на прозрачном
фоне без лишних предметов.

Этот эталон будет использоваться для генерации
фото и видео контента.

💰 Стоимость: {price}₽
💳 Ваш баланс: {balance}₽

Создать эталон?"

[Inline: ✅ Создать эталон]
[Inline: ❌ Отмена | ← Назад | 🏠 Меню]
```

**Действия:**
- Читаем прайслист: `price = get_price('create_reference')`
- Читаем баланс пользователя: `balance = get_user_balance(user_id)`
- Если `balance >= price` → ждём подтверждения
- Если `balance < price` → Шаг 9а (недостаточно средств)

**Переходы:**
- «❌ Отмена» → Шаг 4
- «← Назад» → Шаг 5 (или Шаг 8б-1 если артикул существующий)
- «🏠 Меню» → Шаг 2
- «✅ Создать» → Шаг 10

### Шаг 10. Создание эталона (T2T → I2I)

**Действия:**
1. Проверка баланса: `balance = get_user_stats(user_id)`
2. Если `balance < REFERENCE_COST` → показываем ошибку и выходим
3. T2T API → генерация промпта + категории (`reference_t2t.py`)
4. Коллаж из 3 фото → загрузка в Telegram → получение URL
5. I2I API → генерация эталона (`reference_i2i.py`)
6. Скачивание результата → списание баланса → сохранение в БД
7. Показ финального результата пользователю

**Экран в процессе:**
```
[Баннер + caption]
"Шаг 10 из N: Генерация эталона

⏳ Генерирую эталон для артикула {articul}...
Это может занять 1-2 минуты."
```

**Экран ошибки (недостаточно средств):**
```
[Баннер + caption]
"❌ Недостаточно средств.

💰 Стоимость создания эталона: {REFERENCE_COST}₽
💳 Ваш баланс: {balance}₽

Пополните баланс и попробуйте снова."
```

**Экран результата:**
```
[Фото эталона (отправлено отдельным фото) + caption]
"🎉 Эталон готов!

📦 Артикул: {articul}
📸 Это ваш {ref_number}-й эталон для этого товара
🏷 Категория: {category}

💰 Списано: {REFERENCE_COST}₽
💳 Ваш баланс: {new_balance}₽

Теперь вы можете генерировать фото и видео!"

[Inline:
  📸 Генерировать фото  |  🎥 Генерировать видео
]
[Inline:
  📂 Мои эталоны  |  🏠 Меню
]
```

**Переходы:**
- «📸 Генерировать фото» → Flow генерации фото
- «🎥 Генерировать видео» → Flow генерации видео
- «📂 Мои эталоны» → Flow просмотра эталонов
- «🏠 Меню» → Шаг 2

### Шаг 11. Утверждение эталона

```
[Баннер + фото эталона + caption]
"Шаг 11: Утверждение эталона

🎉 Эталон готов!

📦 Артикул: {articul}
📸 Это ваш {ref_number}-й эталон для этого товара

Проверьте качество:"

[Inline: ✅ Подтвердить | 🔄 Перегенерировать (-{price}₽)]
[Inline: ← Назад | 🏠 Меню]
```

**Переходы:**

#### 11а. 🔄 Перегенерировать
- Проверяем баланс
- Если `balance >= price` → списываем → Шаг 10 (новая генерация)
- Если `balance < price` → Шаг 9а (недостаточно средств)
- Новый эталон получает `ref_number = N + 1`

#### 11б. ✅ Подтвердить → Шаг 12 (сохранение)

- «← Назад» → возврат к Шагу 9
- «🏠 Меню» → Шаг 2

---

### Шаг 12. Сохранение эталона

**Действия:**
```
article_references:
  - user_id
  - articul
  - reference_number = COUNT(active refs for this articul) + 1
  - file_id (telegram file id)
  - file_path (локальный путь)
  - reference_image_url (публичный URL для I2I API)
  - category (из T2T: верх/низ/обувь/головной убор)
  - reference_prompt (английский промпт из T2T)
  - is_active = TRUE
```

| **Экран** | ✅ Да | ❌ Нет | Основной экран с информацией |
| **Промежуточный** | ✅ Да | ✅ Да (сразу) | «⏳ Загружаю...», «⏳ Генерирую...» |
| **Алерт** | ❌ Нет | ✅ Да (5 сек) | Ошибки, предупреждения |

### Сущности БД

| Таблица | Описание |
|---------|----------|
| `users` | Пользователь (balance, is_registered) |
| `articles` | Распарсенные товары (name, color, material, wb_images) |
| `article_references` | Эталоны (N на артикул, ref_number, is_active) |
| `generation_tasks` | Очередь задач (photo/video, status, result_url) |
| `marketplace_cache` | Кэш маркетплейса по артикулу |
| `user_actions` | Лог действий пользователя |

### Прайслист (пример)

| Действие | Цена |
|----------|------|
| Создание эталона | 50₽ |
| Генерация фото | 30₽ |
| Генерация видео | 80₽ |
| Перегенерация эталона | 50₽ |

### Категории товаров

- `верх` — рубашка, футболка, блузка, свитер, куртка
- `низ` — юбка, брюки, шорты, джинсы
- `обувь` — кроссовки, туфли, сапоги
- `головной убор` — шапка, кепка, шляпа
