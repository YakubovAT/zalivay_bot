# Как работает web/

## Стек

- **FastAPI** (Python) — бэкенд, запускается через `uvicorn web.app:app`
- **asyncpg** — подключение к PostgreSQL (пул открывается при старте через `lifespan`)
- **Jinja2** — шаблонизатор для единственной HTML-страницы (`templates/index.html`)
- **Pillow** — генерация превью изображений
- Фронтенд — чистый JS без фреймворков, встроен прямо в HTML

---

## Переменные окружения (`.env`)

| Переменная        | Описание                                                            |
|-------------------|---------------------------------------------------------------------|
| `BOT_TOKEN`       | Токен бота — используется для проверки подписи Telegram             |
| `BOT_USERNAME`    | Username бота без `@` (для виджета входа)                           |
| `WEB_SECRET`      | Секрет для подписи сессионных куков                                 |
| `MEDIA_ROOT`      | Путь к папке с медиафайлами (по умолчанию `./media`)                |
| `DATABASE_URL`    | Строка подключения к PostgreSQL                                     |
| `ADMIN_USER_IDS`  | Telegram user_id через запятую — владельцы с доступом к админ-панели |
| `ASSETS_ROOT`     | Путь к папке с баннерами (по умолчанию `./assets`)                  |

---

## Авторизация

1. Незалогиненному пользователю показывается экран входа с **Telegram Login Widget**.
2. После входа виджет вызывает JS-функцию `onTelegramAuth(user)`.
3. Фронтенд делает `POST /api/auth` с данными от Telegram.
4. Бэкенд проверяет подпись (`HMAC-SHA256` от `BOT_TOKEN`) и актуальность данных (не старше 24 часов).
5. При успехе выдаётся **httponly-куки `session`** — подписанный токен формата `user_id:first_name:timestamp:sig`.
6. Куки живёт **30 дней**. При каждом запросе парсится и проверяется.
7. Выход — `POST /api/logout` — удаляет куки.

## API-роуты

| Метод    | Путь                           | Описание                                                       |
|----------|--------------------------------|----------------------------------------------------------------|
| `GET`    | `/`                            | Главная страница (Jinja2-шаблон)                               |
| `POST`   | `/api/auth`                    | Авторизация через Telegram Login Widget                        |
| `POST`   | `/api/logout`                  | Выход, удаление куки                                           |
| `GET`    | `/api/files`                   | Список медиафайлов текущего пользователя (JSON)                |
| `GET`    | `/api/references`              | Активные эталоны пользователя (не в корзине)                   |
| `PATCH`  | `/api/references/{ref_id}`     | Обновление редактируемых полей эталона                         |
| `DELETE` | `/api/references/{ref_id}`     | Перемещение эталона в корзину (soft delete)                    |
| `GET`    | `/api/trash`                   | Эталоны в корзине + авто-финализация просроченных              |
| `POST`   | `/api/trash/{ref_id}/restore`  | Восстановление эталона из корзины                              |
| `DELETE` | `/api/trash/{ref_id}`          | Окончательное удаление из корзины (необратимо)                 |
| `GET`    | `/thumb/{path}`                | Превью фото 400×400 (генерируется и кэшируется)                |
| `GET`    | `/files/{path}`                | Оригинальный файл (фото, видео или эталон)                     |
| `GET`    | `/api/admin/prompts`                       | Все шаблоны и элементы списков (только admin)  |
| `PATCH`  | `/api/admin/prompts/templates/{key}`       | Сохранить текст шаблона (только admin)         |
| `POST`   | `/api/admin/prompts/lists/items`           | Добавить элемент в список (только admin)       |
| `PATCH`  | `/api/admin/prompts/lists/items/{item_id}` | Обновить элемент списка (только admin)         |
| `DELETE` | `/api/admin/prompts/lists/items/{item_id}` | Удалить элемент списка (только admin)          |
| `POST`   | `/api/admin/prompts/invalidate`            | Сбросить кэш промптов (только admin)           |
| `GET`    | `/api/admin/users`                         | Список всех пользователей с балансами (admin)  |
| `PATCH`  | `/api/admin/users/{user_id}`               | Обновить баланс пользователя (только admin)    |
| `GET`    | `/assets/{name}`                           | Превью баннера из папки assets/ (только admin) |
| `GET`    | `/api/admin/banners`                       | Список файлов баннеров в assets/ (только admin)|
| `POST`   | `/api/admin/banners/{name}`                | Заменить файл баннера (multipart, только admin)|

Все роуты кроме `/` и `/api/auth` требуют авторизации. Доступ к чужим данным исключён: путь к файлу всегда начинается с `user_id`, который сверяется с сессией; все мутирующие запросы проверяют `WHERE user_id = $1` на уровне SQL.

### Редактируемые поля эталона (`PATCH /api/references/{ref_id}`)

Принимаются только поля из белого списка — остальные игнорируются:

| Поле                 | Назначение                                         |
|----------------------|----------------------------------------------------|
| `product_name`       | Название товара                                    |
| `product_color`      | Цвет                                               |
| `product_material`   | Материал / состав                                  |
| `product_description`| Описание на английском (используется при генерации)|
| `category`           | Категория (верх / низ / обувь / головной убор)     |

## Файловая структура media

```
media/
  {user_id}/
    WB/
      {articul}/
        1.webp             ← оригинальные фото с Wildberries (скачиваются при парсинге)
        2.webp
        ...
    references/
      {articul}_ref_final.png   ← сгенерированный эталон
    generated/
      {articul}/
        image.png          ← сгенерированное фото
        .thumb_image.jpg   ← превью (не попадает в листинг)
        video.mp4
```

- `_list_user_files()` сканирует `media/{user_id}/generated/`
- `_find_wb_original()` ищет первый файл в `media/{user_id}/WB/{articul}/` — используется как оригинал товара в разделе Эталоны
- Файлы, начинающиеся с `.thumb_`, пропускаются
- Пути в БД хранятся в формате `media/{user_id}/...`; `_db_path_to_serve_path()` конвертирует их в формат для `/files/` (без префикса `media/`)

## Превью (`/thumb/`)

- Генерируется из оригинала при первом обращении
- Размер: **400×400 px** (thumbnail с сохранением пропорций)
- Сохраняется рядом с оригиналом как `.thumb_{name}.jpg`
- Пересоздаётся автоматически если оригинал новее превью

---

## Корзина эталонов

Мягкое удаление реализовано через столбец `deleted_at TIMESTAMPTZ NULL` в таблице `article_references`.

| Состояние                                | `is_active` | `deleted_at`   |
|------------------------------------------|-------------|----------------|
| Активный эталон                          | `TRUE`      | `NULL`         |
| В корзине                                | `TRUE`      | дата удаления  |
| Окончательно удалён (>30 дней в корзине) | `FALSE`     | дата удаления  |

- `DELETE /api/references/{id}` — проставляет `deleted_at = NOW()`
- `GET /api/trash` — перед выборкой финализирует записи старше 30 дней (`is_active = FALSE`), затем возвращает оставшиеся с полем `days_left`
- `POST /api/trash/{id}/restore` — сбрасывает `deleted_at = NULL`

## Фронтенд (index.html)

Одна страница с двумя состояниями.

### Незалогинен
Карточка с Telegram Login Widget.

### Залогинен — четыре вкладки (четвёртая только для admin)

#### Вкладка «Медиафайлы»
1. При загрузке страницы делает `GET /api/files` — получает список файлов и артикулов.
2. Строит **фильтры** — по типу (Все / Фото / Видео) и по артикулу.
3. Рендерит **сетку карточек** (`grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))`).
4. В сетке — лёгкие превью (`/thumb/`), при клике открывается **лайтбокс** с оригиналом (`/files/`).
5. Лайтбокс поддерживает навигацию стрелками на клавиатуре и кнопками `‹ ›`.

#### Вкладка «Эталоны»
1. При первом открытии параллельно делает `GET /api/references` и `GET /api/trash`.
2. Рендерит **список карточек** (не сетку).
3. Каждая карточка:
   - Слева — два фото рядом: **Оригинал** (первое WB-фото) и **Эталон** (сгенерированный).
   - Справа — поля товара: артикул (только чтение), номер эталона, дата, редактируемые поля и кнопка 🗑.
4. **Inline-редактирование**: клик на значение → `contenteditable` → Enter или потеря фокуса → `PATCH`. Escape — отмена.
5. **Кнопка 🗑** — подтверждение → `DELETE /api/references/{id}` → карточка исчезает, вкладка Корзина появляется.

#### Вкладка «Корзина»
- Скрыта когда пуста, появляется автоматически после первого удаления или если при загрузке в корзине есть записи.
- Каждая карточка показывает фото эталона, артикул, название и счётчик "удалится через N дн." (красный при ≤7 днях).
- Кнопка **Восстановить** — `POST /api/trash/{id}/restore` → эталон возвращается в список, вкладка скрывается если корзина опустела.
- Кнопка **✕** — подтверждение → `DELETE /api/trash/{id}` → окончательное удаление (необратимо), вкладка скрывается если корзина опустела.

#### Вкладка «Администратор» (только для admin)

Видна только пользователям из `ADMIN_USER_IDS`. Состоит из **sidebar-навигации** (сворачивается кнопкой ☰) и двух разделов.

**Sidebar**
- Кнопка ☰ сворачивает/разворачивает панель навигации (ширина 200px, CSS-transition).
- Разделы: ✏️ Промпты, 💬 Сообщения, 👥 Пользователи. Активный раздел подсвечивается акцентным цветом.
- Секции «Сообщения» и «Пользователи» загружаются лениво — только при первом переходе.

**Раздел «Промпты»**

*Шаблоны промптов*
- 8 карточек (4 фото + 4 видео). Ключи фиксированы, редактируется только текст.
- Подсказки с `{placeholder}`-именами выводятся под каждым полем — это контракт с кодом, переименовывать запрещено.
- Кнопка «Сохранить» активируется только при изменении текста → `PATCH /api/admin/prompts/templates/{key}`.

*Списки элементов*
- 9 групп (локации, одежда, цвета). Каждая группа сворачивается/разворачивается.
- Для групп с `value2` (`video_locations`, `video_bottom_items`, `video_top_items`) отображается второй столбец.
- Чекбокс `is_active` — скрывает элемент без удаления → `PATCH` по blur/change.
- Кнопка «+ Добавить» → `POST /api/admin/prompts/lists/items` → строка появляется в DOM без перезагрузки.
- Кнопка «✕» — подтверждение → `DELETE /api/admin/prompts/lists/items/{id}`.

*Кнопка «Сбросить кэш»*
- `POST /api/admin/prompts/invalidate` → сбрасывает кэш в процессе web-сервиса.
- Бот (`zalivai-bot`) использует отдельный in-memory кэш с TTL=10s (в разработке; перед продом — 30–60s) и подхватывает изменения автоматически.

**Раздел «Сообщения»**
- Карточки UI-сообщений бота (`msg_*` ключи из `prompt_templates`).
- Новые мигрированные сообщения появляются автоматически без изменений кода веб-панели.
- Каждая карточка содержит **блок баннера**: превью текущего изображения (`/assets/{name}`) и кнопку «Заменить баннер».
- Замена баннера: выбор файла → `POST /api/admin/banners/{name}` (multipart) → превью обновляется без перезагрузки.
- Бот подхватывает новый файл автоматически при следующей отправке (кэш по mtime, без рестарта).
- Связь баннера с сообщением хранится в поле `banner` таблицы `prompt_templates`.

**Раздел «Пользователи»**
- Таблица всех пользователей: ник (`@username`), Telegram ID, баланс, статус регистрации, дата.
- Поиск в реальном времени — фильтрует строки таблицы по нику или ID.
- **Inline-редактирование баланса**: клик на значение → ввод → Enter или потеря фокуса → `PATCH /api/admin/users/{user_id}`. Escape — отмена.
- Редактируется только баланс; остальные поля доступны только для просмотра.
