# Как работает web/

## Стек

- **FastAPI** (Python) — бэкенд, запускается через `uvicorn web.app:app`
- **asyncpg** — подключение к PostgreSQL (пул открывается при старте через `lifespan`)
- **Jinja2** — шаблонизатор для единственной HTML-страницы (`templates/index.html`)
- **Pillow** — генерация превью изображений
- Фронтенд — чистый JS без фреймворков, встроен прямо в HTML

---

## Переменные окружения (`.env`)

| Переменная     | Описание                                                |
|----------------|---------------------------------------------------------|
| `BOT_TOKEN`    | Токен бота — используется для проверки подписи Telegram |
| `BOT_USERNAME` | Username бота без `@` (для виджета входа)               |
| `WEB_SECRET`   | Секрет для подписи сессионных куков                     |
| `MEDIA_ROOT`   | Путь к папке с медиафайлами (по умолчанию `./media`)    |
| `DATABASE_URL` | Строка подключения к PostgreSQL                         |

---

## Авторизация

1. Незалогиненному пользователю показывается экран входа с **Telegram Login Widget**.
2. После входа виджет вызывает JS-функцию `onTelegramAuth(user)`.
3. Фронтенд делает `POST /api/auth` с данными от Telegram.
4. Бэкенд проверяет подпись (`HMAC-SHA256` от `BOT_TOKEN`) и актуальность данных (не старше 24 часов).
5. При успехе выдаётся **httponly-куки `session`** — подписанный токен формата `user_id:first_name:timestamp:sig`.
6. Куки живёт **30 дней**. При каждом запросе парсится и проверяется.
7. Выход — `POST /api/logout` — удаляет куки.

## API-роуты

| Метод    | Путь                         | Описание                                                       |
|----------|------------------------------|----------------------------------------------------------------|
| `GET`    | `/`                          | Главная страница (Jinja2-шаблон)                               |
| `POST`   | `/api/auth`                  | Авторизация через Telegram Login Widget                        |
| `POST`   | `/api/logout`                | Выход, удаление куки                                           |
| `GET`    | `/api/files`                 | Список медиафайлов текущего пользователя (JSON)                |
| `GET`    | `/api/references`            | Список эталонов пользователя с данными о товаре (JSON)         |
| `PATCH`  | `/api/references/{ref_id}`   | Обновление редактируемых полей эталона                         |
| `GET`    | `/thumb/{path}`              | Превью фото 400×400 (генерируется и кэшируется)                |
| `GET`    | `/files/{path}`              | Оригинальный файл (фото, видео или эталон)                     |

Все роуты кроме `/` и `/api/auth` требуют авторизации. Доступ к чужим данным исключён: путь к файлу всегда начинается с `user_id`, который сверяется с сессией; при `PATCH` проверка `WHERE user_id = $1` на уровне SQL.

### Редактируемые поля эталона (`PATCH /api/references/{ref_id}`)

Принимаются только поля из белого списка — остальные игнорируются:

| Поле                 | Назначение                                         |
|----------------------|----------------------------------------------------|
| `product_name`       | Название товара                                    |
| `product_color`      | Цвет                                               |
| `product_material`   | Материал / состав                                  |
| `product_description`| Описание на английском (используется при генерации)|
| `category`           | Категория (верх / низ / обувь / головной убор)     |

## Файловая структура media

```
media/
  {user_id}/
    WB/
      {articul}/
        1.webp             ← оригинальные фото с Wildberries (скачиваются при парсинге)
        2.webp
        ...
    references/
      {articul}_ref_final.png   ← сгенерированный эталон
    generated/
      {articul}/
        image.png          ← сгенерированное фото
        .thumb_image.jpg   ← превью (не попадает в листинг)
        video.mp4
```

- `_list_user_files()` сканирует `media/{user_id}/generated/`
- `_find_wb_original()` ищет первый файл в `media/{user_id}/WB/{articul}/` — используется как оригинал товара в разделе Эталоны
- Файлы, начинающиеся с `.thumb_`, пропускаются
- Пути в БД хранятся в формате `media/{user_id}/...`; `_db_path_to_serve_path()` конвертирует их в формат для `/files/` (без префикса `media/`)

---

## Превью (`/thumb/`)

- Генерируется из оригинала при первом обращении
- Размер: **400×400 px** (thumbnail с сохранением пропорций)
- Сохраняется рядом с оригиналом как `.thumb_{name}.jpg`
- Пересоздаётся автоматически если оригинал новее превью

## Фронтенд (index.html)

Одна страница с двумя состояниями.

### Незалогинен
Карточка с Telegram Login Widget.

### Залогинен — две вкладки

#### Вкладка «Медиафайлы»
1. При загрузке страницы делает `GET /api/files` — получает список файлов и артикулов.
2. Строит **фильтры** — по типу (Все / Фото / Видео) и по артикулу.
3. Рендерит **сетку карточек** (`grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))`).
4. В сетке — лёгкие превью (`/thumb/`), при клике открывается **лайтбокс** с оригиналом (`/files/`).
5. Лайтбокс поддерживает навигацию стрелками на клавиатуре и кнопками `‹ ›`.

#### Вкладка «Эталоны»
1. Загружается лениво — `GET /api/references` вызывается только при первом переходе на вкладку.
2. Рендерит **список карточек** (не сетку).
3. Каждая карточка:
   - Слева — два фото рядом: **Оригинал** (первое WB-фото) и **Эталон** (сгенерированный).
   - Справа — поля товара: артикул (только чтение), номер эталона, дата, и редактируемые поля.
4. **Inline-редактирование**: клик на значение → `contenteditable` → Enter или потеря фокуса → `PATCH /api/references/{id}`. Escape — отмена без сохранения.
5. После успешного сохранения кратко показывается «Сохранено», при ошибке — «Ошибка сохранения» с восстановлением прежнего значения.
