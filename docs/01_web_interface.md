# Веб-интерфейс (zalivai-web)

## Описание

Веб-интерфейс для просмотра медиафайлов (видео, фото, эталоны) и управления шаблонами сообщений бота.

## Авторизация

**Метод:** Telegram Login Widget (безопасная авторизация через Telegram)

**Поток:**
1. Пользователь видит кнопку "Вход через Telegram"
2. Кликает → открывается окно Telegram
3. Telegram подтверждает личность пользователя и отправляет подпись на сервер
4. Сервер проверяет подпись с помощью BOT_TOKEN
5. Если валидна → создается сессия (session cookie)

**Сессионные куки:**
- **Кодирование:** base64 (для поддержки кириллицы в имени пользователя)
- **Структура токена:** `user_id:first_name:timestamp:signature`
- **Примечание:** Раньше ошибка `UnicodeEncodeError` при имени с русскими буквами — исправлено base64

**Пример вызова:**
```python
# В web/app.py
def _make_session(user_id: int, first_name: str) -> str:
    ts = int(time.time())
    payload = f"{user_id}:{first_name}:{ts}"
    sig = hmac.new(WEB_SECRET.encode(), payload.encode(), hashlib.sha256).hexdigest()[:24]
    token = f"{payload}:{sig}"
    return base64.b64encode(token.encode()).decode()  # <-- base64 кодирование
```

## Переменные окружения

Устанавливаются в `.env`:

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `BOT_TOKEN` | Токен Telegram бота (для проверки подписи) | - |
| `BOT_USERNAME` | Username бота без @ | `ZalivaiBot` |
| `WEB_SECRET` | Секрет для подписи сессионных куков | Любая строка |
| `MEDIA_ROOT` | Путь к папке с медиафайлами | `./media` |
| `ASSETS_ROOT` | Путь к папке с картинками/баннерами | `./assets` |
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql://user:pass@host/db` |
| `ADMIN_USER_IDS` | Telegram user_id админов через запятую | `123456,789012` |

## Права доступа

### Администраторы
- Список в `ADMIN_USER_IDS` (берется из БД или .env)
- Могут редактировать шаблоны сообщений
- Могут управлять эталонами

### Остальные пользователи
- Только просмотр медиафайлов

## Запуск

**На сервере (systemd):**
```bash
systemctl restart zalivai-web
```

**Локально (разработка):**
```bash
uvicorn web.app:app --host 0.0.0.0 --port 8090 --reload
```

## Порт

- **Локально:** 8090
- **На сервере:** 8090 (через systemd)

## Маршруты

| Маршрут | Метод | Описание |
|---------|-------|---------|
| `/` | GET | Главная страница (редиректит на /gallery если авторизован) |
| `/auth` | POST | Обработка Telegram Login Widget (создание сессии) |
| `/logout` | POST | Выход (удаление сессии) |
| `/gallery` | GET | Галерея медиафайлов |
| `/api/media` | GET | API: список медиафайлов |
| `/api/templates` | GET | API: список шаблонов (для админов) |

## БД

Использует ту же БД PostgreSQL что и бот:
- Таблица `users` — сессии хранятся в куках, юзеры в этой таблице
- Таблица `prompt_templates` — шаблоны сообщений
- Таблица `media_items` — медиафайлы (видео, фото)

## Известные проблемы и исправления

### ✅ Исправлено: UnicodeEncodeError в куках

**Проблема:** При попытке авторизации с именем на кириллице (русские буквы) происходила ошибка:
```
UnicodeEncodeError: 'latin-1' codec can't encode characters in position 20-23
```

**Причина:** Starlette использует latin-1 для кодирования cookie значений, но имя пользователя содержало русские буквы.

**Решение:** Base64 кодирование токена перед установкой в cookie.
- **Файл:** `web/app.py`
- **Функции:** `_make_session()`, `_parse_session()`
- **Коммит:** cf89ec2a
