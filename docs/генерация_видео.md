# Генерация видео — полный workflow

> Документ описывает процесс генерации lifestyle-видео на основе эталона товара: от диалога с пользователем до отправки результата.

---

## Диалог с пользователем (flow gen_video)

**Файл:** `handlers/flows/gen_video.py`

### V1. Количество видео

Пользователь вводит число от 1 до 5 вручную или выбирает быстрыми кнопками (1 / 2 / 3).

### V2. Пожелания

Пользователь вводит текст («модель идёт по пляжу») или нажимает «Нет пожеланий».

Если баланс не покрывает стоимость — показывается алерт, flow остаётся на этом шаге.

### V3. Подтверждение

Показывается итог: количество видео, стоимость, баланс. Кнопка «✅ Сгенерировать».

### V4. Очередь

После подтверждения в БД создаётся `generation_job` + N `job_task` с типом `lifestyle_video`. Пользователь видит сообщение «Поставил в очередь». Баланс **ещё не списывается**.

> ⚠️ **Требование к эталону:** поле `product_description` обязательно. Эталоны без него (созданные до внедрения T2T) генерацию видео не поддерживают — бот покажет алерт.

## Формирование промптов

**Файл:** `services/prompt_generator_video.py`

**Функция:** `generate_video_prompts(description, category, count) -> list[str]`

| Аргумент | Источник |
|----------|----------|
| `description` | `article_references.product_description` — готовое EN-описание товара |
| `category` | `article_references.category` — верх / низ / обувь / головной убор |
| `count` | число видео, введённое пользователем |

**Логика:** для каждого видео случайно выбираются локация с описанием движения модели и дополняющий элемент одежды из фиксированных списков. Шаблон подбирается по категории:

| Категория | Шаблон | Дополнение |
|-----------|--------|------------|
| верх | PROMPT_TOP | случайный низ (jeans, trousers…) + цвет |
| низ | PROMPT_BOTTOM | случайный верх (blouse, knit…) + цвет |
| обувь | PROMPT_SHOES | нейтральный аутфит |
| головной убор | PROMPT_HAT | нейтральный аутфит |

**Структура промпта:**
```
A fashion lifestyle video. A model wearing {description},
paired with {item_color} {item}.
Location: {location}.
The model is {motion}.
Smooth cinematic camera movement, natural lighting,
sharp focus on the clothing, professional e-commerce fashion video.
```

Пожелания пользователя дописываются в конец каждого промпта через запятую (в `gen_video.py`).

**Локации с движением** (12 вариантов): city street, coffee shop, park, studio, seaside, rooftop, café, flower market, studio backdrop, pedestrian bridge, forest path, hotel lobby.

## Фоновый воркер

**Файл:** `services/task_worker.py` → `run_video_job_worker()`

- Параллельный, до 5 одновременных задач (asyncio.Semaphore)
- Каждые 5 секунд добирает задачи из очереди `pending`
- Тип задач: `lifestyle_video`
- На каждую задачу: I2V API (`generate_lifestyle_video`) → скачать `.mp4` → сохранить в `media/{user_id}/generated/{article}/`
- При рестарте: зависшие `processing` → `pending` сбрасываются автоматически (окно: 30 мин, дольше чем у фото т.к. видео генерируется дольше)

**Завершение группы (`_finish_video_job`):**

| Итог | Действие |
|------|----------|
| Все задачи упали | Уведомление об ошибке, баланс не трогается |
| Хотя бы одно видео готово | Списывается баланс за успешные видео, отправляется первое видео с подписью |

## I2V API

**Файл:** `services/lifestyle_video_generator.py`

**Функция:** `generate_lifestyle_video(session, api_base, api_key, ref_image_url, prompt) -> str | None`

- Провайдер: `kie.ai`
- Модель: `wan/2.1-i2v-720p` (настраивается через `VIDEO_I2V_MODEL` в `.env`)
- Endpoint: `POST /api/v1/jobs/createTask` → polling `GET /api/v1/jobs/recordInfo`
- Формат результата: `resultJson.resultUrls[0]` — URL `.mp4` файла
- Polling: до 120 попыток × 5 сек = максимум 10 минут ожидания
- Aspect ratio: `2:3` (портретный)

## Тарифы

| Операция | Стоимость |
|----------|-----------|
| Одно видео | `VIDEO_COST` (20₽ по умолчанию, настраивается через `.env`) |

Баланс списывается **только за успешно сгенерированные** видео после завершения всей группы.

## Сводная таблица файлов

| Файл | Ответственность |
|------|-----------------|
| `handlers/flows/gen_video.py` | Диалог с пользователем, создание job + tasks в БД |
| `services/prompt_generator_video.py` | Генерация EN-промптов по description и категории |
| `services/task_worker.py` → `run_video_job_worker` | Параллельный воркер, I2V запросы, отправка результата |
| `services/lifestyle_video_generator.py` | Обёртка над I2V API (kie.ai) |

---

## Сводная таблица БД

| Таблица | Ключевые поля |
|---------|---------------|
| `generation_jobs` | user_id, article, ref_number, ref_image_url, count, cost, screen_msg_id |
| `generation_job_tasks` | job_id, user_id, articul, prompt, status=`lifestyle_video`, result_url, file_path |

Видео-задачи используют те же таблицы что и фото, различаются полем `task_type = 'lifestyle_video'`.
