# Создание видео — полный workflow

> Документ описывает процесс создания lifestyle-видео на основе эталона товара: от диалога с пользователем до отправки результата.

---

## Диалог с пользователем (flow gen_video)

**Файл:** `handlers/flows/gen_video.py`

### V1. Количество видео

Пользователь вводит число от 1 до 5 вручную или выбирает быстрыми кнопками (1 / 2 / 3).

### V2. Пожелания

Пользователь вводит текст («модель идёт по пляжу») или нажимает «Нет пожеланий».

Если баланс не покрывает стоимость — показывается алерт, flow остаётся на этом шаге.

### V3. Подтверждение

Показывается итог: количество видео, стоимость, баланс. Кнопка «✅ Ссоздавать».

### V4. Очередь

После подтверждения в БД создаётся `generation_job` + N `job_task` с типом `lifestyle_video`. Пользователь видит сообщение «Поставил в очередь». Баланс **ещё не списывается**.

> ⚠️ **Требование к эталону:** поле `product_description` обязательно. Эталоны без него (созданные до внедрения T2T) создание видео не поддерживают — бот покажет алерт.

## Формирование промптов

**Файл:** `services/prompt_generator_video.py`

**Функция:** `async generate_video_prompts(description, category, count) -> list[str]`

| Аргумент | Источник |
|----------|----------|
| `description` | `article_references.product_description` — готовое EN-описание товара |
| `category` | `article_references.category` — верх / низ / обувь / головной убор / комбинезон |
| `count` | число видео, введённое пользователем |

**Источник данных:** шаблоны и списки хранятся в БД (таблицы `prompt_templates`, `prompt_list_items`).
Доступ — через `services/prompt_store.py` с TTL-кэшем 60 сек. При недоступности БД — автоматический fallback на значения из кода.

**Логика:** для каждого видео случайно выбирается пара (локация + движение модели) из списка `video_locations`, дополняющий элемент одежды — из `video_bottom_items` / `video_top_items`. Шаблон подбирается по категории:

| Категория | Ключ шаблона | Дополнение |
|-----------|--------------|------------|
| верх | `video_top` | список `video_bottom_items` (item + color) |
| низ | `video_bottom` | список `video_top_items` (item + color) |
| обувь | `video_shoes` | нейтральный аутфит из `video_neutral_outfits` |
| головной убор | `video_hat` | нейтральный аутфит из `video_neutral_outfits` |
| комбинезон | `video_jumpsuit` | нейтральный аутфит из `video_neutral_outfits` |

**Структура промпта** (шаблон `video_top`, остальные аналогичны):
```
Smooth cinematic fashion lifestyle video.
A stylish young woman wearing {description}, paired with {item_color} {item}.
Location: {location}.
The model is {motion}.
Slow gliding camera captures the fabric drape and flow of the garment.
Warm soft natural lighting, cinematic color grading, shallow depth of field.
The top garment stays in sharp focus throughout the motion.
Professional e-commerce fashion footage, no camera shake, fluid movement.
```

**Шаблон `video_jumpsuit`** (полный образ, без дополняющей одежды):
```
Smooth cinematic fashion lifestyle video.
A stylish young woman wearing {description} as a complete look, finished with {outfit}.
Location: {location}.
The model is {motion}.
Slow gliding full-body camera shot captures the silhouette, fabric drape, and movement of the jumpsuit.
Warm soft natural lighting, cinematic color grading, shallow depth of field.
The garment stays in sharp focus throughout the motion — from neckline to hem.
Professional e-commerce fashion footage, no camera shake, fluid movement.
```

> Имена `{placeholder}` в шаблонах — контракт с кодом. Переименовывать их через admin-панель запрещено.

Пожелания пользователя дописываются в конец каждого промпта через запятую (в `gen_video.py`).

**Список `video_locations`** содержит 14 пар (локация, движение): управляется через admin-панель в БД.

## Фоновый воркер

**Файл:** `services/task_worker.py` → `run_video_job_worker()`

- Параллельный, до 5 одновременных задач (asyncio.Semaphore)
- Каждые 5 секунд добирает задачи из очереди `pending`
- Тип задач: `lifestyle_video`
- На каждую задачу: I2V API (`generate_lifestyle_video`) → скачать `.mp4` → сохранить в `media/{user_id}/generated/{article}/`
- При рестарте: зависшие `processing` → `pending` сбрасываются автоматически (окно: 30 мин, дольше чем у фото т.к. видео создается дольше)

**Завершение группы (`_finish_video_job`):**

| Итог | Действие |
|------|----------|
| Все задачи упали | Уведомление об ошибке, баланс не трогается |
| Хотя бы одно видео готово | Списывается баланс за успешные видео, отправляется первое видео с подписью |

## I2V API

**Файл:** `services/lifestyle_video_generator.py`

**Функция:** `generate_lifestyle_video(session, api_base, api_key, ref_image_url, prompt) -> str | None`

- Провайдер: `kie.ai`
- Модель: `sora-2-image-to-video` (настраивается через `VIDEO_I2V_MODEL` в `.env`)
- Endpoint: `POST /api/v1/jobs/createTask` → polling `GET /api/v1/jobs/recordInfo`
- Формат результата: `resultJson.resultUrls[0]` — URL `.mp4` файла
- Polling: до 120 попыток × 5 сек = максимум 10 минут ожидания
- Параметры: `aspect_ratio=portrait`, `n_frames=10`, `remove_watermark=true`, `upload_method=s3`
- Поле изображения в запросе: `image_urls` (массив, max 1 элемент)

## Тарифы

| Операция | Стоимость |
|----------|-----------|
| Одно видео | `VIDEO_COST` (20₽ по умолчанию, настраивается через `.env`) |

Баланс списывается **только за успешно созданные** видео после завершения всей группы.

## Сводная таблица файлов

| Файл | Ответственность |
|------|-----------------|
| `handlers/flows/gen_video.py` | Диалог с пользователем, создание job + tasks в БД |
| `services/prompt_generator_video.py` | Async-создание EN-промптов по description и категории |
| `services/prompt_store.py` | TTL-кэш шаблонов и списков из БД, fallback на хардкод |
| `services/task_worker.py` → `run_video_job_worker` | Параллельный воркер, I2V запросы, отправка результата |
| `services/lifestyle_video_generator.py` | Обёртка над I2V API (kie.ai) |

---

## Сводная таблица БД

| Таблица | Ключевые поля |
|---------|---------------|
| `generation_jobs` | user_id, article, ref_number, ref_image_url, count, cost, screen_msg_id |
| `generation_job_tasks` | job_id, user_id, articul, prompt, status=`lifestyle_video`, result_url, file_path |
| `prompt_templates` | key, template, description, updated_at |
| `prompt_list_items` | list_key, value, value2, is_active, sort_order, updated_at |

Видео-задачи используют те же таблицы что и фото, различаются полем `task_type = 'lifestyle_video'`.
