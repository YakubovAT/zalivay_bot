## 11. ГЕНЕРАЦИЯ 4 ФОТО (services/reference_i2i_welcome.py)

**Входные данные**:
- `image_urls`: первые 4 фото товара
- `category`: категория товара (из T2T)
- `name`: название товара из маркетплейса

**Логика**:

1. **Выбор 4 промтов**:
   - Из БД (таблица `prompt_list_items`) берём 4 случайных промта по категории
   - Пример для категории "низ":
     - Локация (из `list_key = 'photo_locations'`)
     - Детали верха (из `list_key = 'photo_top_items'`)
     - Детали низа (из `list_key = 'photo_bottom_items'`)
     - Стиль/наряд (из `list_key = 'photo_neutral_outfits'`)

2. **Создание I2I запроса**:
   ```
   Создай изображение разделённое на 4 равные части.
   В каждой части новое фото товара.
   
   [ПРОМТ 1]
   [ПРОМТ 2]
   [ПРОМТ 3]
   [ПРОМТ 4]
   
   Аспект 3:4, разрешение 1K, формат PNG (настраиватеся в API Nano Banana 2)
   ```

3. **Получение результата**:
   - I2I возвращает ONE PNG с 4 фото внутри (3:4 formato, 1K quality)

4. **Разделение PNG на 4 части и наложение watermark**:
   - Используем PIL для разрезания одного PNG на 4 отдельных файла
   - Каждый файл сохраняется как оригинал: `photo_{article_code}_{task_id}_1.png`, `photo_{article_code}_{task_id}_2.png` и т.д.
   - На каждый файл накладываем watermark через `services/image_watermark.py` (артикул "арт. XXXXX" + название товара)
   - Сохраняем с watermark: `photo_{article_code}_{task_id}_1_with_text.png`, `photo_{article_code}_{task_id}_2_with_text.png` и т.д.

5. **Сохранение в БД** (таблица `media_files`):
   Для каждого из 4 файлов:
   - `user_id` = ID пользователя
   - `article_code` = артикул товара
   - `file_path` = `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_1.png` (оригинал)
   - `watermarked_path` = `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_1_with_text.png` (с watermark)
   - `result_url` = публичный URL файла с watermark
   - `file_type` = "photo"

6. **Сохранение на диск**:
   
   **Оригиналы (без watermark)**:
   - `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_1.png`
   - `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_2.png`
   - `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_3.png`
   - `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_4.png`
   
   **С watermark (артикул + название товара)**:
   - `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_1_with_text.png`
   - `media/{user_id}/generated/{article_code}/photo_{article_code}_{task_id}_2_with_text.png`

   - Вводить новый артикул (обычный флоу)
   - Просматривать свои предыдущие товары
   - Выбирать фото и создавать эталон (если захочет)

## 15. ОБРАБОТКА ОШИБОК

### Ошибки внешних провайдеров (API):
- **WB API** (парсинг): повторяем запрос до 3 раз с интервалом 2 сек
- **Kie.ai T2T**: повторяем запрос до 3 раз с интервалом 2 сек
- **Kie.ai I2I**: повторяем запрос до 3 раз с интервалом 5 сек

### Ошибки в коде приложения:
- Выводим в лог
- Показываем пользователю: "❌ Что-то пошло не так. Попробуйте позже."
- **Не повторяем** (это уже наша ошибка)

### Валидация данных:
- Если артикул невалидный → ошибка в Шаг 1Ж
- Если парсер вернул пустое имя → ошибка
- Если T2T вернул неизвестную категорию → повторяем запрос
- Если I2I вернул ошибку → повторяем запрос
