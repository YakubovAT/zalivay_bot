# Медиафайлы — хранение, обработка и экспорт

> Документ описывает полный pipeline: авторегистрация сгенерированных файлов → нанесение водяного знака → экспорт в Pinterest CSV.

---

## 1. Реестр медиафайлов (`media_files`)

**Таблица БД:** `media_files`

Каждый созданный ботом файл (фото или видео) автоматически регистрируется в таблице сразу после завершения задачи в воркере.

| Поле | Описание |
|------|----------|
| `id` | PK |
| `user_id` | Владелец |
| `article_code` | Артикул товара |
| `task_id` | FK → `generation_job_tasks.id` (nullable) |
| `file_path` | Абсолютный путь на сервере: `media/{user_id}/generated/{article}/photo_...png` |
| `result_url` | CDN-ссылка от kie.ai (истекает через ~14 дней) |
| `file_type` | `'photo'` или `'video'` |
| `pinterest_export_count` | Сколько раз попал в Pinterest CSV (никогда не исключается) |
| `pinterest_exported_at` | Время последнего экспорта |
| `watermarked_path` | Путь к копии с нанесённым текстом (NULL = ещё не обработано) |
| `created_at` | Время создания |

**Где регистрируется:**

- `services/task_worker.py` → `run_job_worker()` — после `complete_job_task()` вызывает `register_media_file(user_id, articul, task_id, save_path, result_url, "photo")`
- Аналогично в `run_video_job_worker()` для видео

**Публичный URL файла** (для Pinterest CSV):

```
https://zaliv.ai/media/{user_id}/generated/{article_code}/filename.png
```

Строится функцией `_file_path_to_public_url(user_id, file_path)` в `services/pinterest_csv_generator.py` — обрезает `MEDIA_ROOT` и подставляет базовый URL.

## 2. Сервис водяного знака (`/watermark`)

### Алгоритм нанесения

**Файл:** `services/image_watermark.py`

1. Открыть изображение как RGBA
2. Создать прозрачный overlay-слой того же размера
3. Вычислить размер шрифта: `max(20, min(w, h) * 0.045)`, отступ `margin = min(w,h) * 0.035`
4. Случайно выбрать вариант диагонали:
   - **Вариант A:** артикул → правый нижний (поднят на `margin*2`), название → левый верхний (опущено на `margin*2`)
   - **Вариант B:** артикул → левый нижний (поднят на `margin*2`), название → правый верхний (опущено на `margin*2`)
5. Название переносится на несколько строк через `_wrap_text()` — каждая строка не шире `w//2 - margins` (в пределах своей четверти изображения)
6. Нарисовать блоки:
   - Артикул — `_draw_label()` (одна строка)
   - Название — `_draw_label_multiline()` (несколько строк)
   - Фон: `rounded_rectangle` (чёрный, opacity 100/255 ≈ 39%), текст белый
7. `Image.alpha_composite(img, overlay).convert("RGB")`
8. Сохранить в `media/{user_id}/watermarked/{article_code}/{stem}_with_text{ext}` с `quality=95`

**Шрифт** — перебор: DejaVu → Liberation → Ubuntu → FreeSans → macOS Helvetica → macOS Arial Bold → `ImageFont.load_default(size=)`

**Текстовые строки:**
- Артикул: `"арт. {article_code}"`
- Название: берётся с приоритетом: `article_references.product_name` → `articles.name` → артикул. Перенос слов ограничивает ширину блока, обрезка по символам не применяется.

**Публичное API:**

| Функция | Тип | Назначение |
|---------|-----|------------|
| `apply_watermark(file_path, article_code, name, out_path=None)` | sync | PIL-обработка, возвращает путь к копии |
| `apply_watermark_to_media_file(media_file_id, user_id)` | async | читает из БД → вызывает `apply_watermark` → сохраняет `watermarked_path` в БД |

Если `watermarked_path` уже заполнен — файл не перерабатывается, возвращается существующий путь.

### Бот-команда `/watermark`

**Файл:** `handlers/flows/watermark.py`

```
/watermark
  │
  ├─ get_unwatermarked_photos(user_id)
  │     └─ нет фото → "Все фото уже обработаны" → END
  │
  ├─ показать кол-во + кнопки [Обработать N фото] [Отмена]
  │
  ├─ cb_confirm → apply_watermark_to_media_file() для каждого
  │     └─ сообщение "Готово! Обработано: X" (+ "Не удалось: Y" если были ошибки)
  │
  └─ cb_cancel → "Отменено"
```

**Поведение:**
- Обрабатывает только фото (`file_type = 'photo'`), у которых `watermarked_path IS NULL`
- Оригинальные файлы не изменяются — создаётся отдельная копия
- Изображения пользователю не отправляются, только сохраняются на сервере

## 3. Pinterest CSV (`/pinterest`)

**Файл:** `services/pinterest_csv_generator.py`

### Алгоритм генерации

1. Получить все `media_files` пользователя
2. Случайная выборка N строк (N задаёт пользователь)
3. Для каждой строки:
   - Получить `pinterest_settings` (цепочка: артикул → пользователь → дефолт)
   - Сформировать публичный URL (`file_path` → `https://zaliv.ai/media/...`)
   - Для видео: добавить `Thumbnail` (один из ключевых кадров); для фото — пусто
   - Заполнить строку CSV
4. Записать CSV c BOM (`﻿`) для Excel-совместимости
5. Обновить `pinterest_export_count` и `pinterest_exported_at` для каждого файла

**Формат CSV:**

| Поле | Источник |
|------|----------|
| `Title` | `article_name` (обрезается до 100 символов) |
| `Media URL` | `https://zaliv.ai/media/{user_id}/...` |
| `Pinterest board` | `settings["board"]` → `article_name` → `article_code` (fallback-цепочка) |
| `Description` | `settings["hashtags"]` (до 500 символов) |
| `Link` | `settings["link_template"]` с подстановкой артикула |
| `Thumbnail` | Путь к кадру (только для видео, для фото — пусто) |
| `Publish date` | Рандомно с шагом `time_step_min`–`time_step_max` минут, формат `%Y-%m-%dT%H:%M:%S` |

Квотирование: `csv.QUOTE_MINIMAL` (требование Pinterest).

**Настройки экспорта (`pinterest_settings`):**

| Поле | Уровень | Описание |
|------|---------|----------|
| `board` | артикул | Название доски Pinterest |
| `link_template` | пользователь | Ссылка с `{article}` |
| `hashtags` | артикул | Хэштеги в Description |
| `time_step_min/max` | пользователь | Диапазон интервала между пинами (минуты) |

Fallback: если нет настроек уровня артикула → берутся настройки пользователя.

### Бот-команда `/pinterest`

**Файл:** `handlers/flows/pinterest.py`

```
/pinterest
  │
  ├─ показать: баланс, кол-во файлов, цена (1 руб./строка)
  ├─ запросить количество строк (10–200)
  │
  ├─ проверка файлов: файлов < запрошено → предложить использовать имеющееся кол-во
  ├─ проверка баланса: не хватает → предложить макс. доступное кол-во (min 10)
  │
  ├─ кнопка [Создать CSV — X руб.]
  │     └─ generate_pinterest_csv(user_id, rows_count)
  │           └─ deduct_balance(user_id, generated * PINTEREST_CSV_COST)
  │                 └─ отправить файл + подпись "Списано: X руб. | Баланс: Y руб."
  │
  └─ [Отмена]
```

**Стоимость:** `PINTEREST_CSV_COST = 1` рубль за строку (env `PINTEREST_CSV_COST`, дефолт `1`).

Баланс **списывается после** генерации — за фактически попавшие в CSV строки.

## Тестирование

### Предусловие

Перед тестом убедись что у пользователя есть сгенерированные фото:

```
/start → посмотреть статистику → "Создано фото: N" должно быть > 0
```

Если фото нет — сначала создать через `/` (команда создания фото).

---

### Тест 1: `/watermark`

**Сценарий A — есть фото без watermark**

1. Отправить `/watermark`
2. Ожидаемый ответ:
   ```
   Фото без текста: N

   На каждое фото будет нанесено:
   • артикул товара (по диагонали)
   • название товара (по диагонали)

   Оригиналы остаются без изменений.

   [Обработать N фото]  [Отмена]
   ```
3. Нажать **"Обработать N фото"**
4. Ожидаемый ответ: `Готово! Обработано фото: N`

**Проверка файлов на сервере:**
```bash
ssh -i ~/.ssh/id_ed25519_zaliv root@95.211.47.37 \
  "ls /var/www/bots/Zalivai_bot/media/*/watermarked/*/"
```
Должны появиться файлы `*_with_text.png`.

**Сценарий B — все фото уже обработаны**

1. Отправить `/watermark` повторно
2. Ожидаемый ответ: `Все ваши фото уже обработаны — артикул и название нанесены.`

**Сценарий C — нажать "Отмена"**

1. Отправить `/watermark`
2. Нажать **"Отмена"**
3. Ожидаемый ответ: `Отменено.`

### Тест 2: `/pinterest`

**Сценарий A — нормальный flow**

1. Отправить `/pinterest`
2. Бот показывает баланс, количество файлов, цену
3. Ввести число от 10 до 200
4. Нажать **"Создать CSV — X руб."**
5. Ожидаемый результат: `.csv` файл в чат + подпись `Списано: X руб. | Баланс: Y руб.`

**Проверка CSV:**
- Открыть файл — должны быть колонки: `Title, Media URL, Pinterest board, Description, Link, Thumbnail, Publish date`
- `Media URL` — ссылка вида `https://zaliv.ai/media/{user_id}/...`
- `Publish date` — формат `2026-04-25T14:30:00` (без Z)
- Кодировка — UTF-8 с BOM (Excel открывает без кракозябр)
- Загрузить в Pinterest → `Готово! Файл загружен.`

**Сценарий B — недостаточно файлов**

1. Запросить больше строк чем есть файлов
2. Бот предлагает использовать доступное количество

**Сценарий C — недостаточно баланса**

1. Запросить количество строк дороже баланса
2. Бот предлагает максимально возможное (мин. 10)

### Известные ограничения (технический долг)

| # | Проблема | Влияние |
|---|----------|---------|
| 1 | Текст "Все обработаны" показывается и когда фото вообще нет | Вводит в заблуждение |
| 2 | PIL-обработка блокирует event loop бота | При большом числе фото бот тормозит для всех |
| 3 | Нет прогресса при обработке большого числа фото | Пользователь не видит что происходит |
