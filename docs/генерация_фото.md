# Генерация фото — полный workflow

> Документ описывает процесс генерации lifestyle-фото на основе эталона товара: от диалога с пользователем до отправки результата.

---

## Диалог с пользователем (flow gen_photo)

**Файл:** `handlers/flows/gen_photo.py`

### P1. Количество фото

Пользователь вводит число от 1 до 20 вручную или выбирает быстрыми кнопками (1 / 5 / 10).

### P2. Пожелания

Пользователь вводит текст («хочу фото на фоне моря») или нажимает «Нет пожеланий».

Если баланс не покрывает стоимость — показывается алерт, flow остаётся на этом шаге.

### P3. Подтверждение

Показывается итог: количество фото, стоимость, баланс. Кнопка «✅ Сгенерировать».

### P4. Очередь

После подтверждения в БД создаётся `generation_job` + N `job_task`. Пользователь видит сообщение «Поставил в очередь». Баланс **ещё не списывается**.

## Формирование промптов

**Файл:** `services/prompt_generator_cloth.py`

**Функция:** `generate_photo_prompts(description, category, count) -> list[str]`

| Аргумент | Источник |
|----------|----------|
| `description` | `article_references.product_description` — готовое EN-описание товара, сформированное T2T API при создании эталона |
| `category` | `article_references.category` — верх / низ / обувь / головной убор |
| `count` | число фото, введённое пользователем |

**Логика:** для каждого фото случайно выбираются локация и дополняющий элемент одежды из фиксированных списков. Шаблон подбирается по категории:

| Категория | Шаблон | Дополнение |
|-----------|--------|------------|
| верх | PROMPT_TOP | случайный низ (jeans, skirt…) + цвет |
| низ | PROMPT_BOTTOM | случайный верх (blouse, hoodie…) + цвет |
| обувь | PROMPT_SHOES | нейтральный аутфит |
| головной убор | PROMPT_HAT | нейтральный аутфит |

**Структура промпта:**
```
Professional lifestyle fashion photograph.
A model wearing {description}, paired with {item_color} {item}.
Location: {location}.
Natural relaxed pose, high-quality e-commerce photography, realistic lighting, sharp focus on the clothing.
```

Пожелания пользователя дописываются в конец каждого промпта через запятую (в `gen_photo.py`).

## Фоновый воркер

**Файл:** `services/task_worker.py` → `run_job_worker()`

- Параллельный, до 10 одновременных задач (asyncio.Semaphore)
- Каждые 3 секунды добирает задачи из очереди `pending`
- На каждую задачу: I2I API (`generate_lifestyle_photo`) → скачать → сохранить в `media/{user_id}/generated/{article}/`
- При рестарте: зависшие `processing` → `pending` сбрасываются автоматически

**Завершение группы (`_finish_job`):**

| Итог | Действие |
|------|----------|
| Все задачи упали | Уведомление об ошибке, баланс не трогается |
| Хотя бы одно фото готово | Списывается баланс за успешные фото, отправляется первое фото с подписью |

## Сводная таблица файлов

| Файл | Ответственность |
|------|-----------------|
| `handlers/flows/gen_photo.py` | Диалог с пользователем, создание job + tasks в БД |
