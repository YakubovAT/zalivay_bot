# Создание фото — полный workflow

> Документ описывает процесс создания lifestyle-фото на основе эталона товара: от диалога с пользователем до отправки результата.

---

## Диалог с пользователем (flow gen_photo)

**Файл:** `handlers/flows/gen_photo.py`

### P1. Количество фото

Пользователь вводит число от 1 до 20 вручную или выбирает быстрыми кнопками (1 / 5 / 10).

### P2. Пожелания

Пользователь вводит текст («хочу фото на фоне моря») или нажимает «Нет пожеланий».

Если баланс не покрывает стоимость — показывается алерт, flow остаётся на этом шаге.

### P3. Подтверждение

Показывается итог: количество фото, стоимость, баланс. Кнопка «✅ Ссоздавать».

### P4. Очередь

После подтверждения в БД создаётся `generation_job` + N `job_task`. Пользователь видит сообщение «Поставил в очередь». Баланс **ещё не списывается**.

## Формирование промптов

**Файл:** `services/prompt_generator_cloth.py`

**Функция:** `async generate_photo_prompts(description, category, count) -> list[str]`

| Аргумент | Источник |
|----------|----------|
| `description` | `article_references.product_description` — готовое EN-описание товара, сформированное T2T API при создании эталона |
| `category` | `article_references.category` — верх / низ / обувь / головной убор / комплект |
| `count` | число фото, введённое пользователем |

**Источник данных:** шаблоны и списки хранятся в БД (таблицы `prompt_templates`, `prompt_list_items`).
Доступ — через `services/prompt_store.py` с TTL-кэшем 60 сек. При недоступности БД — автоматический fallback на значения из кода.

**Логика:** для каждого фото случайно выбираются локация и дополняющий элемент одежды из списков в БД. Шаблон подбирается по категории:

| Категория | Ключ шаблона | Дополнение |
|-----------|--------------|------------|
| верх | `photo_top` | список `photo_bottom_items` + цвет из `photo_colors` |
| низ | `photo_bottom` | список `photo_top_items` + цвет из `photo_colors` |
| обувь | `photo_shoes` | нейтральный аутфит из `photo_neutral_outfits` |
| головной убор | `photo_hat` | нейтральный аутфит из `photo_neutral_outfits` |
| комплект | `photo_komplekt` | нейтральный аутфит из `photo_neutral_outfits` |

**Структура промпта** (шаблон `photo_top`, остальные аналогичны):
```
Fashion lifestyle editorial photograph.
A stylish young woman wearing {description}, paired with {item_color} {bottom_item}.
Setting: {location}.
Confident, natural relaxed pose.
Soft diffused natural light, warm tones, shallow depth of field with blurred bokeh background.
Sharp focus on the top garment — fabric texture, fit, and drape clearly visible.
Photorealistic commercial photography, high resolution, no distortion.
```

**Шаблон `photo_komplekt`** (полный образ, без дополняющей одежды):
```
Fashion lifestyle editorial photograph.
A stylish young woman wearing {description} as a complete outfit.
Accessories: {neutral_outfit}.
Setting: {location}.
Full-body composition, natural confident pose.
Soft diffused natural light, warm tones, shallow depth of field with blurred bokeh background.
Sharp focus on the jumpsuit — fabric texture, fit, silhouette, and drape clearly visible from neckline to hem.
Photorealistic commercial photography, high resolution, no distortion.
```

> Имена `{placeholder}` в шаблонах — контракт с кодом. Переименовывать их через admin-панель запрещено.

Пожелания пользователя дописываются в конец каждого промпта через запятую (в `gen_photo.py`).

> ⚠️ **Требование к эталону:** поле `product_description` обязательно. Все эталоны, созданные после внедрения T2T (возврат трёх полей: category, prompt_i2i, description), содержат его. Старые эталоны (до этого изменения) — не содержат и создание фото не поддерживают. Таких эталонов в БД на момент проверки: **22 из 23**.

## Фоновый воркер

**Файл:** `services/task_worker.py` → `run_job_worker()`

- Параллельный, до 10 одновременных задач (asyncio.Semaphore)
- Каждые 3 секунды добирает задачи из очереди `pending`
- На каждую задачу: I2I API (`generate_lifestyle_photo`) → скачать → сохранить в `media/{user_id}/generated/{article}/`
- При рестарте: зависшие `processing` → `pending` сбрасываются автоматически

**Завершение группы (`_finish_job`):**

| Итог | Действие |
|------|----------|
| Все задачи упали | Уведомление об ошибке, баланс не трогается |
| Хотя бы одно фото готово | Списывается баланс за успешные фото, отправляется первое фото с подписью |

## Сводная таблица файлов

| Файл | Ответственность |
|------|-----------------|
| `handlers/flows/gen_photo.py` | Диалог с пользователем, создание job + tasks в БД |
| `services/prompt_generator_cloth.py` | Async-создание EN-промптов по description и категории |
| `services/prompt_store.py` | TTL-кэш шаблонов и списков из БД, fallback на хардкод |
| `services/task_worker.py` | Параллельный воркер, I2I запросы, отправка результата |
| `services/lifestyle_photo_generator.py` | Обёртка над I2I API (kie.ai) |
