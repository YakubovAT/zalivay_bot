# Велком Флоу (Шаги 1а-1е + 1ж + Результаты)

## Описание

Новый пользователь проходит полный велком флоу:

**Часть 1: Приветствие (Шаги 1а-1е)**
- **Шаг 1а: Приветствие** — Привет и проблема отсутствия трафика
- **Шаг 1б: Проблема** — Реклама на WB/Ozon как счётчик такси
- **Шаг 1в: Решение** — Пример продавца с трафиком через Pinterest
- **Шаг 1г: Как помогает бот** — ИИ генерирует пины + CSV + база креаторов
- **Шаг 1е: Результаты** — Результаты клиентов и призыв к действию

**Часть 2: Демонстрация (Шаг 1ж + Генерация)**
- **Шаг 1ж: Ввод артикула** — Пользователь вводит артикул WB
- **Парсинг WB** — Система получает название, цвет, материал, фото
- **T2T Генерация** — AI определяет категорию товара и создает описание
- **I2I Генерация** — AI создает эталон товара и 4 фото разных стилей
- **Pinterest CSV** — Автоматическая генерация CSV для загрузки
- **Результаты** — Показываются 4 фото и CSV для скачивания
- **Автопереход** — Через 60 сек переход в главное меню

## Технические детали

### Файлы

**Обработчики:**
- `handlers/flows/onboarding.py` — шаги 1a-1e + routing к welcome_article_input
- `handlers/flows/welcome_article_input.py` — шаги 1ж + парсинг + генерация + результаты

**Клавиатуры:** `handlers/keyboards/__init__.py`
- `kb_welcome_article_input()` — для ввода артикула
- `kb_welcome_csv_ready()` — для результатов CSV

**Сервисы:**
- `services/wb_parser_welcome.py` — парсинг товара из WB
- `services/reference_t2t_welcome.py` — генерация категории + описания
- `services/reference_i2i_welcome.py` — генерация эталона + 4 фото
- `services/image_prompt_generator.py` — генерация lifestyle-промтов (4 раза для велком флоу)
- `services/pinterest_csv_generator.py` — генерация CSV (переиспользуется)

**База данных:**
- `database/schema.sql` — поле `is_welcome` в таблице users
- `database/db.py` — функция `mark_welcome_completed(user_id)`
- Сообщения: `msg_welcome_1a` ... `msg_welcome_1e`, `msg_welcome_step_1f`, `msg_loading_welcome`, `msg_csv_ready_welcome`

### Поток выполнения

```
/start
  ↓
is_welcome = false?
  ├─ Да → Шаги 1a-1e (приветствие)
  │         cb_welcome_start_work() → Шаг 1ж (ввод артикула)
  │         handle_article_input() → Парсинг WB
  │         ↓
  │         T2T: категория + описание
  │         ↓
  │         I2I: эталон + 4 фото
  │         ↓
  │         CSV: генерация для Pinterest
  │         ↓
  │         Показ результатов (4 фото + CSV)
  │         ↓
  │         Ожидание 60 сек
  │         ↓
  │         mark_welcome_completed() → is_welcome = true
  │         ↓
  │         Переход в Меню (Шаг 2)
  │
  └─ Нет → Прямо в Меню (Шаг 2)
```

## Жизненный цикл

1. Пользователь нажимает `/start`
2. Система проверяет `is_welcome` в БД
3. Если `is_welcome = false` → показываются шаги 1a-1e
4. При нажатии "Начать работу" → ввод артикула (шаг 1ж)
5. Ввод артикула → автоматический парсинг + T2T + I2I + CSV
6. Показ результатов → через 60 сек переход в меню + `is_welcome = true`
7. При следующем `/start` → прямо в меню (велком не показывается)
