# Создание эталона — полный workflow

> Документ описывает весь процесс создания эталона товара: от ввода артикула до сохранения результата в БД.

---

## Шаг 1. Ввод артикула

**Файл:** `handlers/flows/new_article.py` → `msg_article_input()`

Пользователь вводит артикул (например `225616209`). Бот извлекает цифры через regex `\d{6,10}`.

**Действия:**
- Отправляется временное сообщение `⏳ Ищу товар...N` с анимацией (макс. 5 сек)
- Вызывается `wb_parser.get_product_info(articul)`

**Результат парсинга WB:**
```
name: "Юбка шорты летняя с воланами и кружевами"
brand: "RILAVIE"
color: "белый" (первый из списка)
material: "полиэстер; хлопок" (очищен от цифр и процентов)
images: список URL фото (до 29 штук)
```

## Шаг 3. Сохранение товара в БД

**Файл:** `handlers/flows/new_article.py` → `cb_product_yes()`

Пользователь нажал "✅ Да, это он".

Вызывается `save_article()`:

```sql
INSERT INTO articles (
    user_id,           -- ID пользователя Telegram
    article_code,      -- "225616209"
    marketplace,       -- "WB"
    name,              -- "Юбка шорты летняя с воланами и кружевами"
    color,             -- "белый"
    material,          -- "полиэстер; хлопок"
    wb_images          -- список всех URL фото товара
) VALUES (...)
```

Данные сохраняются в `context.user_data["product"]`.

## Шаг 4. Выбор 3 фото

**Файл:** `handlers/flows/photo_selection.py` → `start_photo_selection()`

Все фото товара скачиваются в `media/{user_id}/WB/{articul}/`.

Пользователь листает фото, выбирает 3 лучших:
- Выбранные сохраняются в `context.user_data["chosen_photo_paths"]`
- Кнопки: `⚪ 1`, `⚪ 2`, `⚪ 3` → при выборе становятся `🔘 1`, `🔘 2`, `🔘 3`

## Шаг 5. Создание коллажа

**Файл:** `handlers/flows/photo_selection.py` → `cb_photos_confirm()`

## Шаг 6. Проверка баланса

**Файл:** `handlers/flows/create_reference.py` → `start_reference_generation()`

Пользователь нажал "✅ Создать эталон".

Проверяется баланс:
```python
stats = await get_user_stats(user_id)
balance = stats["balance"]

if balance < REFERENCE_COST:  # 5₽
    # Показываем алерт с кнопкой "❌ Закрыть"
    return _REFERENCE_GENERATING
```

Если баланс достаточен → показывается "⏳ Создаю эталон..."

---

## Шаг 7. Создание промпта эталона (T2T API)

**Файл:** `services/reference_t2t.py` → `generate_reference_prompt()`

Делается запрос к T2T API (gpt-5-2) с данными товара:
- name, color, material

**Возвращается 3 поля:**
```json
{
  "category": "низ",
  "prompt_i2i": "Analyze the image and detect a white summer skort with ruffles and lace...",
  "description": "White summer skort made of polyester and cotton with ruffles and lace details"
}
```

| Поле | Назначение |
|------|-----------|
| **category** | Категория товара (верх/низ/обувь/головной убор/комплект) |
| **prompt_i2i** | Промпт для I2I API: изоляция товара, удаление фона/модели |
| **description** | Краткое описание товара на английском (для создания фото/видео в будущем) |

`prompt_i2i` используется сразу для создания эталона, `description` сохраняется в БД для будущего использования.

**При ошибке T2T API:** отправляется новое текстовое сообщение с описанием ошибки и кнопкой **❌ Закрыть** (callback `close_error`). Прогресс-сообщение «⏳ Создаю эталон...» остаётся нетронутым. Flow завершается.

---

## Шаг 8. Подготовка фото для I2I

**Файл:** `handlers/flows/create_reference.py`

Каждое из выбранных оригинальных фото отправляется в Telegram чтобы получить публичный CDN URL:
```python
for path in chosen_paths:
    sent = await context.bot.send_photo(chat_id=user_id, photo=open(path, "rb"))
    file_obj = await context.bot.get_file(sent.photo[-1].file_id)
    image_urls.append(file_obj.file_path)  # → https://api.telegram.org/file/bot.../photos/...
```
Временные сообщения сразу удаляются (пользователь не видит).

---

## Шаг 9. Создание эталона (I2I API)

**Файл:** `services/reference_i2i.py` → `generate_reference_image()`

Запрос к `kie.ai` I2I API (модель `nano-banana-2`):
- `image_input` — список Telegram CDN URL из шага 8 (по одному на каждое фото)
- `prompt` — английский промпт из шага 7
- `aspect_ratio: 9:16`, `resolution: 1K`, `output_format: png`

**API возвращает:** URL созданного эталона

**При ошибке I2I API:** отправляется новое текстовое сообщение с описанием ошибки и кнопкой **❌ Закрыть** (callback `close_error`). Баланс не списывается. Flow завершается.

---

## Шаг 10. Определение номера, скачивание и списание баланса

**Файл:** `handlers/flows/create_reference.py`

- Вычисляется `reference_number`: `get_reference_count(user_id, article) + 1`
  - `get_reference_count` считает только `is_active = TRUE AND deleted_at IS NULL` — эталоны в корзине не учитываются
- Скачивается результат по URL → `media/{user_id}/references/{articul}_ref_{N}.png` (N = reference_number)
  - Каждый эталон хранится в отдельном файле; перезаписи при создании нового эталона нет
- Списывается баланс: `deduct_balance(user_id, REFERENCE_COST)` → 5₽
- Отправляется фото для получения Telegram `file_id`
- Временное сообщение удаляется

---

## Шаг 11. Сохранение эталона в БД

**Файл:** `handlers/flows/create_reference.py`

Эталон сохраняется в таблицу `article_references`:

```sql
INSERT INTO article_references (
    user_id,              -- ID пользователя
    articul,              -- "225616209"
    reference_number,     -- 1 (N-й эталон для этого артикула)
    file_id,              -- Telegram file_id фото
    file_path,            -- "media/{user_id}/references/{articul}_ref_{N}.png"
    reference_image_url,  -- URL эталона от kie.ai
    category,             -- "низ" (из шага 7)
    reference_prompt,     -- prompt_i2i из шага 7 (для создания эталона)
    product_description,  -- description из шага 7 (для создания фото/видео)
    product_name,         -- "Юбка шорты летняя с воланами и кружевами"
    product_color,        -- "белый"
    product_material,     -- "полиэстер; хлопок"
    source_photo_paths    -- JSON-массив путей к 3 исходным фото (для пересоздания)
) VALUES (...)
```

## Шаг 12. Показ результата

**Файл:** `handlers/flows/create_reference.py`

Исходное сообщение редактируется:
```
[Фото эталона (отправлено отдельным фото) + caption]
"Шаг 11 из N: Эталон готов!

📦 Артикул: 225616209
📸 Это ваш 1-й эталон для этого товара
🏷 Тип товара: низ

💰 Списано: 5₽
💳 Ваш баланс: 0₽

Эталон может немного отличаться от оригинала.
Если отличия значительные — пересоздайте эталон,
заменив фотографии на шаге выбора фото.

Теперь вы можете создавать фото и видео!"

[Inline:
  🔄 Переделать эталон
]
[Inline:
  📸 Создать фото  |  🎥 Создать видео
]
[Inline:
  📂 Мои эталоны  |  🏠 Меню
]
```

Кнопка **🔄 Переделать эталон** запускает тот же flow пересоздания (Шаг 16а, `regen_reference`), что и из карточки эталона. Перед показом результата в `user_data` сохраняется `ref_number_for_gen = reference_number`, что позволяет `cb_regen_start` найти нужный эталон.

- Если есть → текст `msg_my_refs_list` с кнопками по артикулам: `📦 {code} ({ref_count})`

## Шаг 14. Карточка эталона (Шаг 16)

**Файл:** `handlers/flows/etalon.py` → `cb_ref_article()`, `cb_ref_nav()`

Пользователь нажал на артикул или листает эталоны (`← Пред. / След. →`).

Показывается фото эталона + caption из `msg_ref_card`:

```
📸 Эталон #1 из 2
📦 Артикул: 225616209
🏷 Тип товара: низ
```

Текст `msg_ref_card` получается через `await msg_ref_card(ref_number, total, article, category)` — async-функция из `handlers/flows/messages/regen_reference.py`, текст берётся из БД (`prompt_store`).

Кнопки: `📸 Создать фото`, `🎥 Создать видео`, `🔄 Переделать эталон`, `← Пред.` / `След. →`, `🏠 Меню`.

---

## Шаг 15. Пересоздание эталона (Шаг 16а)

**Файл:** `handlers/flows/regen_reference.py`

Запускается по кнопке «🔄 Переделать эталон» на карточке эталона.

**Процесс:**

1. `cb_regen_start()` — проверяет наличие исходных фото на диске, показывает запрос пожеланий (`msg_regen_wish`)
2. Пользователь вводит пожелания или нажимает «Пропустить»
3. `_run_regen()`:
   - Проверка баланса → `msg_insufficient_funds` если не хватает
   - Прогресс `msg_regen_generating`
   - T2T API → промпт + категория (как при создании)
   - Пожелания дописываются в конец промпта: `{prompt_i2i}. Additional notes: {wish}`
   - Исходные фото поштучно загружаются в Telegram → I2I API (`nano-banana-2`) → новый эталон
   - Списание баланса, сохранение в `article_references` с новым `reference_number`
   - Показ результата `msg_regen_result`

Если исходные фото не найдены на диске → `msg_regen_no_source_photos`, завершение.

**Все тексты** (msg_regen_wish, msg_regen_generating, msg_regen_result, msg_regen_no_source_photos, msg_ref_card) — async-функции в `handlers/flows/messages/regen_reference.py`, источник — БД через `prompt_store.get_template()`.

## Сводная таблица БД

| Таблица | Поля | Описание |
|---------|------|----------|
| `articles` | user_id, article_code, marketplace, name, color, material, wb_images | Распарсенные товары WB |
| `article_references` | user_id, articul, reference_number, file_id, file_path, reference_image_url, category, reference_prompt, product_description, product_name, product_color, product_material, **source_photo_paths**, **is_active**, **deleted_at** | Эталоны (N на артикул) |
| `users` | id, balance, is_registered, ... | Пользователи |

### Статусная модель `article_references`

| Состояние | `is_active` | `deleted_at` |
|-----------|-------------|--------------|
| Активный (виден в боте и web) | `TRUE` | `NULL` |
| В корзине web (не виден в боте) | `TRUE` | дата удаления |
| Окончательно удалён (>30 дней в корзине) | `FALSE` | дата удаления |

Все бот-запросы (`get_active_references`, `get_user_articles_with_refs`, `get_reference_count`, `get_reference`) фильтруют: `is_active = TRUE AND deleted_at IS NULL`. Эталоны в корзине web в боте не отображаются и не учитываются при нумерации.
