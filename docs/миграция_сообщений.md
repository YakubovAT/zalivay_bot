# Миграция сообщений в БД

> Документ описывает процесс переноса UI-сообщений бота и AI-промптов из хардкода в БД,
> механику работы кэш-слоя и текущий статус миграции по каждому блоку.

---

## Зачем

Чтобы тексты, шаблоны и списки редактировались через admin-панель без перезапуска бота.
После сохранения изменений в БД бот подхватывает их автоматически в течение TTL кэша (10 сек в dev, 30–60 в prod).

---

## Механика

### Хранилище

Два типа данных в одной таблице `prompt_templates`:

| Префикс ключа | Тип | Пример |
|---|---|---|
| `photo_*`, `video_*` | AI-промпты для генерации | `photo_top`, `video_shoes` |
| `msg_*` | UI-сообщения бота пользователю | `msg_welcome` |

Списки элементов (локации, одежда, цвета) — в таблице `prompt_list_items`.

### Доступ из кода

Все обращения — только через `services/prompt_store.py`:

```python
from services.prompt_store import get_template

text = await get_template("msg_welcome", fallback=_FALLBACK_TEXT)
```

**Приоритет:** БД → `fallback`-аргумент → `_FALLBACK_TEMPLATES` в `prompt_store.py` → пустая строка.

### Кэш

- In-memory словарь с TTL
- При истечении TTL — запрос к БД
- При недоступности БД — тихий fallback, бот не падает

### Seed данных

Каждое новое сообщение добавляется в `database/schema.sql` блоком:

```sql
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM prompt_templates WHERE key = 'msg_xxx') THEN
    INSERT INTO prompt_templates (key, template, description) VALUES (...);
  END IF;
END $$;
```

Проверка по конкретному ключу (не по пустой таблице) — безопасно при повторных запусках `init_db()`.

### Что можно редактировать через admin

- Текст шаблона вокруг `{placeholder}`
- Элементы списков (`prompt_list_items`): добавить, отредактировать, отключить (`is_active = FALSE`)

### Что нельзя трогать

- Имена `{placeholder}` — контракт с кодом (иначе бот сломается при подстановке)
- Поле `key` в шаблонах и `list_key` в списках — привязаны к коду

## Статус миграции

### AI-промпты — ✅ выполнено

**Файлы:** `services/prompt_generator_cloth.py`, `services/prompt_generator_video.py`

| Ключ | Категория | Описание |
|------|-----------|----------|
| `photo_top` | Фото — верх | Переменные: `{description}`, `{item_color}`, `{bottom_item}`, `{location}` |
| `photo_bottom` | Фото — низ | Переменные: `{description}`, `{item_color}`, `{top_item}`, `{location}` |
| `photo_shoes` | Фото — обувь | Переменные: `{description}`, `{neutral_outfit}`, `{location}` |
| `photo_hat` | Фото — головной убор | Переменные: `{description}`, `{neutral_outfit}`, `{location}` |
| `video_top` | Видео — верх | Переменные: `{description}`, `{item_color}`, `{item}`, `{location}`, `{motion}` |
| `video_bottom` | Видео — низ | Переменные: `{description}`, `{item_color}`, `{item}`, `{location}`, `{motion}` |
| `video_shoes` | Видео — обувь | Переменные: `{description}`, `{outfit}`, `{location}`, `{motion}` |
| `video_hat` | Видео — головной убор | Переменные: `{description}`, `{outfit}`, `{location}`, `{motion}` |

**Списки элементов:**

| list_key | Назначение |
|---|---|
| `photo_locations` | Локации для фото (12 вариантов) |
| `photo_bottom_items` | Элементы низа для категории «верх» |
| `photo_top_items` | Элементы верха для категории «низ» |
| `photo_neutral_outfits` | Нейтральные аутфиты для обуви и шапок |
| `photo_colors` | Цвета дополняющей одежды |
| `video_locations` | Локации + движение модели (пары value / value2) |
| `video_bottom_items` | Низ для категории «верх» (пары item / color) |
| `video_top_items` | Верх для категории «низ» (пары item / color) |
| `video_neutral_outfits` | Нейтральные аутфиты для видео |

## Сводная таблица файлов

| Файл | Ответственность |
|---|---|
| `database/schema.sql` | Создание таблиц + seed-данные (idempotent) |
| `services/prompt_store.py` | TTL-кэш, `get_template()`, `get_list()`, `get_pairs()`, `get_banner()`, `invalidate()` |
| `services/prompt_generator_cloth.py` | Async-генератор фото-промптов, читает из store |
| `services/prompt_generator_video.py` | Async-генератор видео-промптов, читает из store |
| `handlers/flows/onboarding.py` | Шаг 1 и шаг 2 — читает `msg_welcome`, `msg_profile` и баннер из store |
| `handlers/flows/messages/common.py` | Общие UI-сообщения, включая `msg_profile` (форматирование и подстановка) |
| `handlers/flows/flow_helpers.py` | Отправка экранов; кэш баннеров по mtime (перечитывает файл при замене) |

## Сводная таблица БД

| Таблица | Ключевые поля |
|---|---|
| `prompt_templates` | key (PK), template, description, banner, updated_at |
| `prompt_list_items` | id, list_key, value, value2, is_active, sort_order, updated_at |
